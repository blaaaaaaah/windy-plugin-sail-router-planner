# Plan 02-01: Timeline Interaction & Enhanced Forecast Data

## Objective
Transform the static forecast table into an interactive timeline-based tool with comprehensive sailing data matching user requirements.

## Task Breakdown

### Task 1: Timeline Slider Component
**Goal**: Create left-side timeline for forecast navigation
**Files**: `src/components/TimelineSlider.svelte`
**Implementation**:
- Vertical slider positioned left of forecast table
- Time markers with hour/date labels
- Drag handle for scrubbing through forecast
- Emit timeline position changes to parent
- Responsive design for mobile/desktop

**Acceptance Criteria**:
- ✅ Timeline displays 24-48 hour range based on forecast data
- ✅ Smooth dragging with position indicator
- ✅ Time labels show readable format (e.g., "14:00", "Mar 12")
- ✅ Emits current time position for forecast updates

### Task 2: Route Progress Visualization
**Goal**: Show boat position along route during timeline scrub
**Files**: `src/components/RouteProgress.svelte`, `src/utils/RouteProgressUtils.ts`
**Implementation**:
- Calculate boat position based on current timeline time
- Visual indicator on map showing predicted position
- Smooth animation between positions during scrubbing
- Integration with existing route display

**Acceptance Criteria**:
- ✅ Boat position calculated accurately based on leg speeds and timeline
- ✅ Position indicator visible on map during timeline interaction
- ✅ Smooth transitions when scrubbing through time
- ✅ Position resets appropriately when route or speeds change

### Task 3: Editable Departure Time & Leg Speeds
**Goal**: Make departure time and leg speeds editable with instant updates
**Files**: `src/components/ForecastTable.svelte` (enhanced)
**Implementation**:
- Inline editing for departure time (datetime input)
- Inline editing for each leg speed (number input with validation)
- Debounced updates to prevent excessive API calls
- Visual feedback during updates

**Acceptance Criteria**:
- ✅ Departure time editable with datetime picker or input
- ✅ Leg speeds editable per route segment
- ✅ Changes trigger forecast recalculation within 2 seconds
- ✅ Invalid inputs show appropriate validation messages

### Task 4: Wave Period Data Integration
**Goal**: Add wave period to existing wave data in forecast
**Files**: `src/types/WeatherTypes.ts`, `src/services/WeatherForecastService.ts`
**Implementation**:
- Extend WeatherData interface to include wave period
- Update Windy API integration to fetch wave period
- Display wave period alongside wave height in forecast table
- Ensure wave period has appropriate units and formatting

**Acceptance Criteria**:
- ✅ Wave period data retrieved from Windy API
- ✅ Wave period displayed in forecast table with appropriate units (seconds)
- ✅ Wave period formatted consistently with other data
- ✅ No breaking changes to existing wave height display

### Task 5: True/Apparent Wind Toggle
**Goal**: Allow switching between true and apparent wind data display
**Files**: `src/components/ForecastTable.svelte` (enhanced)
**Implementation**:
- Toggle switch for True Wind vs Apparent Wind
- Recalculate all wind-related data when toggled
- Update column headers and data display
- TODO: Store preference for user session (future settings panel)

**Acceptance Criteria**:
- ✅ Toggle clearly labeled (True Wind / Apparent Wind)
- ✅ All wind data (speed, direction, gusts) switches appropriately
- ✅ Column headers update to reflect current mode
- ✅ Calculations accurate for both true and apparent wind

### Task 6: Enhanced Weather Metrics & Statistics
**Goal**: Add comprehensive wind statistics per leg like PredictWind
**Files**: `src/types/WeatherTypes.ts`, `src/services/WeatherForecastService.ts`
**Implementation**:
- Extend WeatherData interface with statistical fields
- WeatherForecastService calculates aggregated statistics per leg
- Add min/max/average wind speeds
- Add wind direction statistics
- Add gust statistics and passage time
- Format statistics in user-friendly way

**Acceptance Criteria**:
- ✅ Min/Max/Average wind speeds calculated per leg by WeatherForecastService
- ✅ Gust statistics (max gust speed) included in WeatherData
- ✅ Passage time displayed per leg in readable format
- ✅ Statistics update when route or speeds change

## Integration Points
1. **Timeline ↔ Forecast Table**: Timeline position drives forecast display
2. **Editable Fields ↔ Weather Service**: Changes trigger forecast recalculation
3. **Route Progress ↔ Map**: Position indicator synchronized with map display
4. **Wind Toggle ↔ All Wind Data**: Affects calculations throughout forecast

## Testing Strategy
- **Manual Testing**: Interactive timeline scrubbing with visual verification
- **Edge Cases**: Very short/long routes, extreme weather conditions
- **Performance**: Smooth scrubbing with large forecast datasets
- **Mobile**: Touch interaction with timeline on mobile devices

## Risk Mitigation
- **Performance**: Debounce timeline updates to prevent excessive calculations
- **Data Accuracy**: Validate wind statistics calculations against known scenarios
- **UX Complexity**: Progressive disclosure of advanced features
- **API Limits**: Cache calculations to minimize Windy API calls during editing
